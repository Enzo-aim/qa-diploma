## 2. Перечень используемых инструментов с обоснованием выбора

- IntelliJ IDEA - Бесплатная среда разработки с открытым исходным кодом. Она предоставляет базовые функции для
  написания, редактирования, отладки и анализа кода.
- Docker - Для быстрой разработки, тестирования и развёртывания приложений благодаря контейнеризации. Он позволяет "
  упаковать" приложение со всеми его зависимостями (код, библиотеки, инструменты) в контейнер.
- Java - Для создания кроссплатформенных приложений, которые могут работать на различных устройствах и операционных
  системах. Он широко применяется для разработки веб-приложений.
- Git - Cистема контроля версий, которая отслеживает изменения в файлах проекта, позволяя работать над кодом в команде и
  возвращаться к предыдущим версиям при необходимости
- JUnit5 - Это фреймворк для написания и запуска автоматизированных модульных тестов на языке Java. Он позволяет
  разработчикам проверять корректность работы отдельных частей кода.
- Selenide - Это фреймворк для автоматизированного тестирования веб-приложений, который является надстройкой (оберткой)
  над Selenium WebDriver. Он упрощает написание тестов благодаря лаконичному синтаксису, автоматическому решению проблем
  с AJAX и ожиданиями, а также другим встроенным функциям, таким как автоматическое создание скриншотов и закрытие
  браузера.
- GitHub - Это онлайн-платформа, основанная на системе контроля версий Git, которая позволяет разработчикам хранить свои
  проекты, отслеживать изменения в коде и совместно работать над ним
- Gradle - Это система автоматической сборки, которая автоматизирует процесс создания приложений. Она компилирует
  исходный код, управляет зависимостями (библиотеками), запускает тесты и упаковывает проект в исполняемый файл.
- Allure - Это фреймворк для создания отчетов о тестировании программного обеспечения.


## 3. Перечень и описание возможных рисков при автоматизации

- Изменения структуры API или приложения может привести к несовместимости автотестов.
- Отсутствие доступа к банковским сервисам может принести может к невозможности проведения автотестов.
- Неправильное конфигурация тестового окружения может привести к некорректным результатам автотестов.
- Отсутствие тестовых меток у html элементов что вызывает сложность определения правильных селекторов для использования
  в автотестов.


## 4. Интервальная оценка с учётом рисков в часах
   - Позитивные сценарии: 15 часов.
   - Негативные сценарии: 18 часов.
   - Настройка окружения и подготовка тестовых данных: 6 часов.
   - Разработка автотестов: 24 часов.
   - Отладка и исправление ошибок: 8 часов.
